# PRD Step 3: 사용자 프로필 및 레시피 저장

## 1. 목적
사용자가 추천받은 레시피를 보관하고 개인 취향에 맞는 경험을 제공합니다.

## 2. 주요 기능
- **사용자 프로필**: 계정 생성, 선호 요리 종류 설정, 알레르기 식재료 설정.
- **레시피 북 (My Recipes)**: 마음에 드는 레시피를 저장하고 언제든 다시 볼 수 있는 기능.
- **활동 내역**: 최근에 시도한 요리 및 저장한 레시피 이력 관리.

## 3. 기술 스택 및 데이터 관리
- **Database**: Local Storage (초기 단계) 또는 간단한 Backend DB (Supabase/Firebase 등).
- **Authentication**: JWT 또는 소셜 로그인 (필요 시).
- **Storage**: JSON 형식으로 저장된 개인별 레시피 목록.

## 4. 사용자 흐름 (User Flow)
1. 사용자가 마음에 드는 레시피 화면에서 '저장하기' 아이콘 클릭.
2. '내 레시피' 탭으로 이동하여 저장된 요리 목록 확인.
3. 프로필 설정에서 '못 먹는 재료(예: 오이, 견과류)'를 등록하면 이후 레시피 생성 시 AI가 제외함.

## 5. 성공 지표
- 재방문율 (저장된 레시피를 다시 보기 위해 방문하는 횟수)
- 개인화 만족도 (알레르기 재료 제외 정확도 100% 목표)
- 데이터 동기화 안정성
